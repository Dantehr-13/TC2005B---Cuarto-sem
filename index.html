INTRODUCCIÓN A LAS APLICACIONES WEB

Dante Hernández Ramírez - A01668070

¿Cuál es la diferencia entre internet y la World Wide Web?

El internet principalmente está compuesto de infraestructura física, protocolos, sistemas de nombres de dominio y la red mundial (WWW). En pocas palabras, la World Wide Web está contenida dentro de lo que es internet.

¿Cuáles son las partes de una URL?
  • Protocolo
  • Subdominio
  • Dominio
  • Subcarpeta
  • Slug
  • Parámetros
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Solicita un recurso, como un HTML del servidor web
  • HEAD: Solicita solo el encabezado del archivo, sin el mismo
  • POST: Se utiliza para enviar un recurso en específico
  • PUT: Similar a POST, con la diferencia de que este puede ser ejecutado varias veces
  • PATCH: Modificaciones parciales de un recurso específico
  • DELETE: Elimina un recurso específico
¿Qué método HTTP se debe utilizar al enviar un formulario HTML?

POST, ya que los datos que se registran se envían en el cuerpo del HTTP.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET, ya que es el principal método de obtención de datos y recursos.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto?

El código de respuesta 200 indica que la solicitud de datos fue exitosa, siendo la respuesta estándar en páginas web.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 404?

El error 404 indica que, aunque el servidor logró establecer conexión exitosamente, el recurso no se ha encontrado. Esto podría suceder por varias razones, en algunas de ellas, como por ejemplo la eliminación de la página o algún enlace roto, se responsabiliza al desarrollador por no encontrar los recursos.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 500?

El código 500 (Internal Server Error) indica un fallo inesperado en el servidor que impide obtener el recurso solicitado por el usuario.

¿Qué significa que un atributo HTML5 esté depreciado (deprecated)?

Es una advertencia de que el atributo está obsoleto. Su uso ya no se recomienda y posiblemente se eliminará a futuro.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
  • Soporte multimedia nativo
  • Elementos semánticos
  • Gráficos y animaciones
  • Bases de datos
  • Documento más ligero
¿Qué componentes de estructura y estilo tiene una tabla?
  • Filas
  • Columnas
  • Celdas organizadas
¿Cuáles son los principales controles de una forma HTML5?
  • Entradas de texto y datos
  • Selección y rango
  • Elementos de selección y texto largo
  • Botones
¿Qué tanto soporte HTML5 tiene el navegador que utilizas?

Mi navegador (Google Chrome) tiene una calificación de 526 puntos, teniendo un promedio de calificación aceptable, teniendo como aspecto crítico los componentes web, en donde carece de HTML imports y Elementos personalizables.

¿Cuál es el ciclo de vida de los sistemas de información?
  • Planificación
  • Análisis
  • Diseño
  • Desarrollo
  • Implementación
  • Mantenimiento
  • Retirada
¿Cuál es tu recomendación sobre el uso de !important en CSS?

Siendo un método que rompe con los ciclos de CSS con el fin de imponer el propio, no es una acción tan recomendable, ya que impone su prioridad por sobre los otros estilos del documento.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

En un aspecto de diseño, afecta principalmente la percepción de la página, el como es percibida por la gente. Además, el elegir una imagen poco favorable puede derivar en problemas de legibilidad y rendimiento.

¿Cuál es tu recomendación al elegir las unidades de una propiedad de estilo entre %, px y pt?

Siendo ambas medidas disponibles para el mismo uso, realmente en situaciones en donde no sea un requisito la exactitud del formato del tamaño del texto, es indiferente. En otro caso, en donde se solicita una medida exacta, la precisión de los pixeles es más confiable.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Mejora el rendimiento precisamente por la disminución del tamaño del archivo, eliminando todos los caracteres innecesarios, espacios en blanco, comentarios entre otras cosas sin alterar su funcionalidad original.

¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Java es un lenguaje compilado y orientado a objetos, mientras que JavaScript es un lenguaje interpretado basado en prototipos, funcional principalmente en desarrollo web.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5)
  • new Date(): fecha y hora actual.
  • getFullYear(): Obtiene el año.
  • getMonth(): Mes.
  • getDay(): Día de la semana.
  • getTime(): Timestamp en milisegundos.
¿Qué métodos tienen los arreglos? (Menciona al menos 5)
  • push(): Añade uno o más elementos al final de un arreglo
  • pop(): Elimina el último elemento de un arreglo
  • shift(): Elimina el primer elemento de un arreglo y lo devuelve
  • map(): Crea un nuevo arreglo con los resultados de aplicar una función a cada uno de los elementos.
  • filter(): Crea un nuevo arreglo con todos los elementos que cumplan la condición implementada.
¿Cómo se declara una variable con alcance local dentro de una función?

Se define directamente dentro del cuerpo de una función o bloque de código.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Disminuye la legibilidad, dificulta la depuración y reduce la modularidad de código.

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Las aplicaciones no funcionarán correctamente si los datos de los usuarios se almacenan en el formato incorrecto. Además, se puede mejorar la experiencia del usuario al proporcionar retroalimentación inmediata sobre los errores. Se puede obligar a los usuarios a introducir contraseñas seguras que faciliten la protección de sus datos.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Se puede lograr de varias formas, siendo la más rudimentaria el bloque de JavaScript del navegador. Otra forma es interceptar la petición HTTP que hace la forma al servidor y modificar los datos antes de que lleguen al servidor.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
